# Milestone 0 Proof Procedure

Run `npm run probe` on the target Windows machine. The command performs passive executable/version/help/authentication discovery and local adversarial broker/filesystem/environment proofs, then writes a redacted report under `evidence/local/`. When both adapters normalize as `SUBSCRIPTION_VERIFIED`, the same command automatically runs disposable active provider fixtures for schema enforcement, exact/latest resume, repository-instruction suppression, broker-only writes, and cancellation. It never attempts active turns when either subscription prerequisite fails.

Passive flag discovery is not behavioral proof. Active fixture turns prove structured schema enforcement, exact/latest behavior, marker suppression for personal/repository configuration, broker-only tools, read-only provider confinement with MCP-scoped writes, and process-tree cancellation. Name-based resume remains capability-probed and optional until its provider exposes a stable noninteractive naming surface. Subscription-limit fingerprints are recorded when observed without intentionally consuming a user's subscription. Active proofs use disposable repositories and contain no personal source.

- [Codex non-interactive mode](https://developers.openai.com/codex/noninteractive/)
- [Codex CLI reference](https://developers.openai.com/codex/cli/reference/)

The gate passes only if both adapters normalize `SUBSCRIPTION_VERIFIED` and active evidence proves customization isolation, broker-only command access, and outside-worktree denial. Missing/unrunnable/unauthed providers are prerequisites, not evidence of an architectural defect. A provider that is available and authenticated but cannot meet an isolation or confinement proof causes `FAIL_ARCHITECTURE` and stops later milestones.

The Windows capability proof passed on 2026-07-14 with Codex 0.144.4 and Claude Code 2.1.209 after recording current subcommand flag placement, direct-root output schemas, Zod/MCP schema compatibility, Codex broker approval semantics, Claude's capability-dependent Worker isolation profile, and bundled MCP startup behavior. This is local proof evidence, not a claim that future CLI versions are compatible without re-probing.
